## 2026-05-02

### 新增命令
* 新增 `python tools/run.py android test-debug`，统一从仓库根触发 Android debug JVM unit tests，对应 Gradle 任务 `:app:testDebugUnitTest`，并写入 Android 命令组帮助说明。

### 工作流调整
* 调整 `python tools/run.py android strings-add` 默认流程：旧口径会把英文 key 同步写入所有已有 `values-*` 文件，新口径只写英文 `values/` 基线，然后自动运行 `tools/scripts/android/translate/run.py key-alignment --json-output` 生成 `temp/translation_key_alignment_reports/`。
* 调整 Android translation `add-key` 契约：`--localized` 不再是默认 fallback，而是仅用于品牌名、协议 token、不可翻译 UI 符号等明确全语言共享文本；JSON 输出新增 `localized_fallback_used`，方便 agent 判断是否走了例外路径。
* 更新 `tools/scripts/android/translate/AGENTS.md`，新增 Android XML key flow，要求新增 key 先走 `python tools/run.py android strings-add`，再按 key-alignment 任务报告和 locale profile 补齐各语言。

### 校验与策略
* 扩展 `tools/scripts/android/translate/run.py key-alignment`，新增 `--res-dir` 与 `--output-dir` 参数，让 agent job 和测试 fixture 可以在临时资源目录中生成独立报告。
* 更新 translation CLI contract 与 key-alignment 文档，明确 `add-key` 的退出码、英文基线默认行为、显式 localized fallback 的适用边界，以及新增 key 后应通过报告修复本地化。
* 扩展 Android translate 单元与 flow 测试，覆盖 `add-key` 默认只写英文基线、显式 `--localized` 才写本地化、`key-alignment --res-dir --output-dir --json-output` 生成结构化缺译报告。

### 修复
* 修正 Android runner 中 staging/release APK 输出路径提示，使 `python tools/run.py android assemble-staging` 和 `assemble-release` 查找 `FlipBits-staging.apk` / `FlipBits-release.apk`，与 Android 构建产物命名保持一致。
