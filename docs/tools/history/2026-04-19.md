## 2026-04-19

### 新增命令
* 新增 `python tools/run.py message prep`，默认优先读取已落盘的 history 文件生成 `temp/message.txt`，把 commit message 草稿链路固定为“先 history、后 message”。
* 新增 `python tools/run.py file-name prep`，可对当前 git 改动中的新增/重命名文件做保守机械检查，并把 agent 可继续改写的草稿写到 `temp/file-name/message.txt`。

### 工作流调整
* 调整 `python tools/run.py` 的帮助与命令组织，根命令现在只展示主要命令组，`history`、`message`、`file-name`、`artifact`、`android` 等能力统一按主命令/子命令层级下钻。
* 调整 release 文档工作流，`history` 与 `message` 现在拆成两条独立链路；提交信息默认以上游已落盘 history 为准，而不是重新从 git 改动独立重建语义。
* 调整 `temp/` 的草稿使用口径，命名检查与 commit message 草稿都统一落到仓库根 `temp/`，避免中间产物散落。

### 校验与策略
* 更新 `.agent` 下的 git message 与 file-name 规则口径，明确哪些规则适合机械检查，哪些必须保留给 agent 做最终判断。
* 更新工具入口文档与总说明，将“先看主命令 `--help`，再按子命令继续下钻”固定为当前仓库 Python CLI 的默认使用方式。

### 重构
* 重构 `tools/wavebits_tools` 到 `tools/repo_tooling`，把内部包名切到职责更清晰的新口径，并同步收口 CLI、policy、history prep 等实现路径。
* 重构 `history prep`、CLI 分组与 retirement/policy 相关实现，把大文件拆到按职责分层的模块，降低后续继续扩展工具工作流时的维护成本。

### 修复
* 修复工具帮助输出过长、顶层噪音过高的问题，当前 `tools/run.py` 顶层帮助已收敛为主命令地图，减少 agent 读取无关参数时的上下文浪费。
* 修复 message 草稿与 history 草稿相互脱节的问题，当前 commit message workflow 已默认消费 history 结果，不再和 release-history workflow 并行漂移。
