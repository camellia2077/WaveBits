## [v0.2.2] - 2026-04-26

### 新增功能 (Added)
* 新增 CLI `encode --flash-style` 参数，支持 `coded_burst`、`ritual_chant` 与 `deep_ritual` 三种 `flash` 风格，并把所选风格写入 WAV metadata，方便命令行产物与 Android 播放侧保持一致的 flash 发声语义。
* 新增 `windows build --build-tests` 与 `configure --build-type / --build-tests` 入口，让 Windows 交付构建可以显式选择 `Debug/Release` 与是否保留 `Test/` 目标，而不必手工改 CMake cache。

### 技术改进/重构 (Changed/Refactor)
* 重构 Rust CLI 的 `flash` 编解码配置链路，将 `FlashStyle` 抽成共享枚举并贯通 `cli`、`bag_api`、`audio_io_api` 与 metadata 读写，统一 flash 风格在命令行帮助、编码配置和解码回放中的语义。
* 重写 `encode --help` 的模式说明与示例文案，明确 `flash / pro / ultra` 的输入边界、ASCII 约束与风格参数，让顶层帮助回到轻量总览，详细说明收敛到子命令帮助。
* 调整 Windows 构建工作流，默认在交付构建里关闭 `Test/` 目标、清理残留 `test_*.exe`，并在 CLI 产物旁自动复制所需运行时 DLL，减少手工整理发布目录的步骤。

### 修复 (Fixed)
* 修复 CLI 无法表达 Android 新增 flash 风格的问题，避免命令行生成的 WAV metadata 继续丢失 `ritual_chant` / `deep_ritual` 风格信息。
* 修复非 `flash` 模式误带风格参数时的配置漂移风险，确保 `pro` 与 `ultra` 仍回落到稳定的默认 flash metadata 行为。
* 修复 Windows 交付构建目录容易混入旧测试可执行文件和缺失运行时 DLL 的问题，让 `FlipBits.exe` 导出结果更接近可直接分发的目录状态。
