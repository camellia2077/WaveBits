## [v0.2.0] - 2026-04-19

### 新增功能 (Added)
* 新增独立 `Rust CLI` 主线：`apps/audio_cli/rust` 现提供基于 `clap` 的 `version` / `encode` / `decode` 命令，并成为当前 CLI presentation 的正式入口。
* 新增真实 `WAV` 编解码链路：CLI 现通过 `bag_api` 复用 text `<->` PCM、通过 `audio_io` C ABI 复用 mono PCM16 WAV 读写，不再停留在最初的 stub artifact 骨架。

### 技术改进/重构 (Changed/Refactor)
* 重构 CLI 构建入口，把 Rust CLI 接入仓库主构建；根仓 `CMake` 现在会桥接 Cargo 构建 CLI，`python tools/run.py verify --build-dir build/dev` 也会把 `cargo test` 纳入默认验证链路。
* 重构 CLI 代码组织，把 `apps/audio_cli/rust/src/lib.rs` 周边拆成 `cli` / `commands` / `bag_api` / `audio_io_api` / `fs_io` / `error` / `util` 等独立文件，降低 presentation 层继续扩展命令和 FFI 时的耦合度。
* 重构 CLI 命令结构与帮助文案，使 `version` / `encode` / `decode` 保持主命令层级，并把帮助输出收敛为“先看主命令、再按子命令 `--help` 下钻”的更可读形态。

### 修复 (Fixed)
* 修复 `decode` 对显式 `--mode` 的依赖：CLI 现在改为从 `WAV metadata` 自动恢复 transport mode / frame samples，避免 encode 后 decode 还需要用户手工重复输入模式信息。
* 修复 CLI 与仓库工作流脱节的问题：旧的 C++/Windows CLI 入口已退出主线，Rust CLI 现在与仓库 `build` / `verify` / smoke 产物链路保持一致，减少 presentation 侧“单独可跑、主构建不认”的漂移。
