## [v0.2.3] - 2026-04-30

### 新增功能 (Added)
* 新增 CLI `mini` 模式，接入库层 Morse code transport：`FlipBits encode --mode mini` 现在可生成 Morse-compatible 文本音频，并与 Android 使用同一套 `mini / flash / pro / ultra` 模式顺序。
* 新增 CLI 对 `mini` WAV metadata 的读写支持，decode 已保存音频时可以识别 `mini` 模式，而不是只能识别 `flash / pro / ultra`。
* 新增 Flash 四情绪 style 参数：`--flash-style` 现在支持 `steady`、`hostile`、`litany`、`collapse`，与 Android 的 Flash 风格选择器和 WAV metadata 取值保持一致。
* 新增 CLI 帮助文案中的 `mini` 模式说明和 Flash 四情绪说明，示例中加入 `FlipBits encode --text "PRAISE THE OMNISSIAH" --mode mini --out out.wav`。

### 技术改进/重构 (Changed/Refactor)
* 重构 Rust FFI 常量，跟随 C API 的新 transport mode 与 Flash voicing style 枚举，移除旧 `coded_burst / ritual_chant / deep_ritual` 映射。
* 更新 CLI metadata writer 到 `WBAG` metadata v6 布局，补齐 `sample_rate_hz`、`payload_byte_count` 与 `input_source_kind` 字段，使 CLI 保存的 WAV 与 Android / libs 的 metadata contract 对齐。
* 调整默认 Flash style 为 `steady`；非 `flash` 模式仍会回落到稳定默认值，避免 `mini / pro / ultra` 误写 Flash 风格语义。
* 更新项目 README / README_en 的模式速览，补充 `mini` Morse code、Flash 四风格频率和 Morse speed preset 说明，方便 CLI 用户从根文档理解新模式。

### 修复 (Fixed)
* 修复 CLI 仍引用旧 Flash style 名称的问题，`encode --help`、命令解析测试和集成测试现在都使用 `steady / hostile / litany / collapse`。
* 修复 Rust `audio_io` metadata FFI 布局落后于 native C ABI 的问题，避免 decode 或 metadata roundtrip 在 v6 字段上错位。

### 弃用/删除 (Deprecated/Removed)
* 删除 CLI 用户可见的旧 Flash style 参数 `coded_burst`、`ritual_chant`、`deep_ritual`；命令行统一使用新的四情绪 style。
