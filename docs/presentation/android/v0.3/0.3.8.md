## [v0.3.8] - 2026-04-28

### 新增功能 (Added)
* 新增长文本音频的文件缓存生成链路，Android 侧会把超出单段容量的输入按 UTF-8 字节预算分段编码，再把生成后的 PCM、follow timeline 与 waveform preview 组合成同一个播放结果，使千字符级输入完成后也能正常进入 mini player。
* 新增长音频播放与导出对文件型 PCM 的支持，生成音频、保存音频与导出流程可以在不把完整 PCM 常驻 UI state 的情况下复用同一份缓存音频。
* 新增 Android 调试辅助脚本 `tools/scripts/android/logcat_extract_crash.py`，便于从 `adb logcat` 中提取长音频生成、解码与崩溃相关日志。

### 技术改进/重构 (Changed/Refactor)
* 重构播放可视化数据模型，使用 `PlaybackPcmVisualData` 显式区分完整 PCM 与 waveform preview，避免上层通过 sample 数量猜测数据语义。
* 抽出 `SegmentedEncodeAudioAssembler`、`GeneratedAudioCacheGateway` 与 `PcmFileAudioUtils`，把长文本分段编码、PCM 文件写入、WAV 导出和预览采样从 Compose UI state 中拆出。
* 重构 mini player / 详情页的可视化入口，使用 `PlaybackVisualizationRoute` 与 `FlashSignalVisualizationInput` 统一选择短音频 PCM 分析、长音频 follow timeline buckets 与 fallback 波形预览，让 `FSK lanes`、`Tone energy`、`Visual` 与 `Lyrics` 复用同一套显示边界。
* 拆分 Android native JNI bridge 为编码、生命周期与公共 helper 文件，降低长文本编码路径继续扩展时单文件膨胀的维护成本。

### 修复 (Fixed)
* 修复长文本生成进度到 `100%` 后不显示 mini player 的问题，生成完成后会落盘缓存并用最新生成音频刷新当前播放源。
* 修复长文本音频进入 mini player 后 `FSK lanes` / `Tone energy` 卡住、只显示局部波峰或无法随播放时间推进的问题，长音频现在可用 follow timeline 生成能量 buckets 来驱动原有可视化 UI。
* 修复长文本音频详情页 `Lyrics` 显示 payload unavailable、文本不随播放时间高亮的问题，file-backed 生成结果会异步补齐 follow 数据并恢复 token 跟随显示。
* 删除 `Input text` 卡片中面向用户的 `over 512 bytes` 自动分段提示及所有 locale 残留翻译，让长文本分段保持为内部实现细节。
