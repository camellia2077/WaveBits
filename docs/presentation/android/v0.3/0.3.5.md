## [v0.3.5] - 2026-04-26

### 新增功能 (Added)
* 新增按阵营主题分流的输入示例库，为 `Ancient Dynasty`、`Sacred Machine`、`Immortal Rot`、`Scarlet Carnage`、`Exquisite Fall` 与 `Labyrinth of Mutability` 提供独立的 `short/long` 示例文本，并让 `dual-tone` 主题切换时自动同步到对应阵营语气。
* 新增 `French` 与 `Korean` 两套 Android 语言资源，并把阵营示例文本一并落到当前维护的本地化目录中，补齐 `values-fr`、`values-ko` 与现有多语言样例文件集。

### 技术改进/重构 (Changed/Refactor)
* 重构输入示例状态为 `SampleFlavor + SampleInputLength` 组合，统一 `AudioTabScreen`、`AudioInputEditorDialog` 与 `AndroidSampleInputTextProvider` 的取样入口，避免输入卡片、编辑器弹窗与主题切换各自维护一套样例选择逻辑。
* 调整 Android 语言口径，将葡萄牙语资源从 `values-pt` 明确收口到 `values-pt-rBR` / `pt-BR`，并同步更新语言枚举、Gradle locale 过滤、字符串一致性校验与翻译脚本的目录识别规则。
* 更新 `tools/scripts/android/translate/` 工作流，为 `compare_translation_quality`、`check_mixed_language`、`apply_translation_replacements` 与统一入口 `run.py` 补齐当前语言集合和 Android XML 资源约束，减少多语言样例批量维护时的人工整理成本。

### 修复 (Fixed)
* 修复 `Edit input text` 弹窗中的骰子随机入口，让它和输入卡片共用当前 `short/long` 状态，避免在 `short` 长度下抽到 `long` 示例，或在编辑器里出现与主卡片不一致的样例文本。
* 修复 `Edit input text` 顶栏骰子图标未接入 `dual-tone` 主题色的问题，使返回按钮、随机按钮与 `Done` 操作统一走主题 accent token。
* 修复 `values-fr`、`values-ko`、`values-uk` 及 `pt-BR` 相关资源中的 XML closing tag、ASCII 引号/撇号与非法转义问题，并补上默认 `values/strings.xml` 中缺失的 `audio_action_clear_result_busy`，恢复 `assembleDebug` 构建通过。
