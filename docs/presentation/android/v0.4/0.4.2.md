## [v0.4.2] - 2026-05-02

### 新增功能 (Added)
* 新增 Flash `Visual` 下方的 8-bit readout：播放时会按上一组与当前组两行展示 binary bits，并随播放进度逐步 reveal，帮助用户对照波形写出 bit 序列。
* 新增 Lyrics/Follow 的 `Tokenizer` 抽屉入口：用户可以从播放器详情页打开完整 token 列表，点击 token 卡片后直接跳转到对应起始 sample，避免在横向 token strip 中滑动预览和点击跳转互相干扰。
* 新增 `mini` Morse decode 结果视图：`Read result` 现在在 `mini` 模式下提供 Morse tab，只显示可复制的 Morse code，不再在 `Slow / Standard / Fast` 与 `Short / Long` 之间重复展示输入原文。
* 新增 `mini` / `pro` 输入框下方的实时编码可用性提示，明显非法输入会禁用 encode，并提供 `Encoding rules` 详情入口说明各模式可接受的字符范围。

### 技术改进/重构 (Changed/Refactor)
* 重构 Flash `Visual` 为固定 segment timeline：Flash 优先使用 `followData.binaryGroupTimeline` 生成稳定 low/high segment，并按绝对 sample 映射到画布坐标，播放时只移动 playhead/viewport，避免短 bit 在慢放时持续变形。
* 优化 Flash `Visual` 三种视图的表现边界：`Lanes` 改为离散协议视图，`Energy` 改为能量仪表视图，`Pitch` 改为 true segmented bars，并加入 micro bars、active edge glow 与 perceptual hold，让短 low/high 视觉上更稳、更容易区分。
* 优化 Lyrics display line 生成逻辑：播放跟随先按 24/32 display units 拆成 UI display lines，再渲染 active line 和上下文，长 ASCII、Morse 与 CJK 文本都能统一换行，不再只依赖 CJK 才避免右侧裁剪。
* 调整 Flash voicing 选择器为单行配置入口，并移除独立 `Signal profile` UI；当前只保留 `Voicing style` 作为高度耦合的风格切换入口，选项文案统一为名词口径，如 `Stasis`、`Hostility`。
* 调整 `mini` Morse token 卡片展示方式，将每个字母与对应 Morse group 组合展示，并用竖向分隔线加强字母组边界，提升 Lyrics 模式下 dot/dash 与原字符的对应关系。
* 调整 Android 构建产物命名，`debug`、`staging`、`release` APK 会在 assemble 后重命名为 `FlipBits-<variant>.apk`。

### 修复 (Fixed)
* 修复 `mini` visual 播放到末尾后不继续滑动时图像被横向拉伸的问题，timeline window 现在保持固定 sample 宽度。
* 修复 `Read result` 折叠状态下仍显示 `READ` 按钮的问题，折叠时结果操作区会一起收起。
* 修复播放器 info 弹窗内容过多时不可滚动的问题，文件详情现在可以纵向滚动查看完整信息。
