## [v0.4.3] - 2026-05-03

### 新增功能 (Added)
* 新增 Flash `Zeal` / `Void` 风格选项，并同步 Android native package、JNI wire value、保存音频 metadata 和多语言文案，让用户可以在 Audio 页选择新的高密度/低频慢速 Flash preset，保存后也能恢复对应风格。
* 新增播放器 Audio info 中的 Flash signal 明细：Flash 生成结果现在会展示 low/high carrier、bit duration、payload silence 和 decode path，这些信息由 core 的真实 profile/layout 派生，避免 Android 前端维护重复规则。
* 新增 WAV header probe 到 Android 音频 I/O 链路，保存音频和未保存生成结果都能复用 sample rate、duration、file size 等真实 WAV 信息；文件大小和 payload 大小统一以 `MB` 显示。

### 技术改进/重构 (Changed/Refactor)
* 调整 Android 端 Flash 风格描述和状态模型以匹配库层 `Steady`、`Hostile`、`Zeal`、`Void` 的最新 timing/voicing 语义，`Hostility` 文案不再描述为复用 steady shell。
* 重构保存音频加载/选择流程，选择 saved audio 时会派生并缓存 WAV info 与 Flash signal info，播放器详情页、底部控制区和保存音频列表不再各自推导大小、时长或 Flash 参数。
* 拆分 Android `audio_io_jni.cpp` 的数组、字符串、枚举映射、DTO marshalling 和 JNI entrypoint 职责，降低 WAV metadata / probe / decode 边界继续扩展时的 native 文件复杂度。
* 拆分 Audio 输入卡、Flash preset picker、Lyrics/Follow token 卡片和 raw annotation blocks，让 screen 层更偏向渲染组合，UTF-8 byte mapping、hex nibble、Morse group 等显示映射进入独立 helper。
* 更新 `apps/audio_android/AGENTS.md` 的 native/JNI 验证提示，并把 Android presentation versionName 提升到 `0.4.3`。

### 修复 (Fixed)
* 修复保存音频文件大小只从 MediaStore 或估算值读取的问题，导入和播放已保存 WAV 时优先使用 native probe 返回的实际 `fileByteCount`。
* 修复播放器信息弹窗缺少 Flash 生成参数的问题，生成后未保存音频和加载后的 saved audio 都会沿用同一套 cached info。
* 补充 codec、decode、player detail 和 UI state 测试，覆盖 Flash signal info、WAV info 缓存和 `MB` 显示口径。
