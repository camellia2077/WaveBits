## [v0.4.11] - 2026-05-14

### 技术改进/重构 (Changed/Refactor)
* 将底部主 tab 重做为更明显的胶囊式 dock 样式，收紧 `Audio / Saved / Settings` 三个入口的容器、选中指示和 mini player 底座的一体化视觉，让底部导航不再像默认 Material tab，而更接近播放器本身的 chrome。
* 扩展 Flash 播放展示链路，播放器现在支持 `Visual / Mix / Lyrics` 三种显示模式，并将 `Pulse / Pitch / Lanes` 收敛到同一套播放竖线与 bit readout 口径，方便在 mini player 和 detail sheet 中直接观察 bit 对齐。
* 重构 Flash visual 与 token 跟随呈现：`Pulse` 去掉 low/high 切换竖线和左侧 `0/1` 标签，`Lanes` 改为更贴 bit span 的实心矩形，token `hex/bin` 改为始终显示完整 byte，仅对当前 nibble / bit 做高亮，减少 visual、`8 bits` 行与 token readout 之间的语义偏差。
* 将播放器倍速控制从连续滑杆收敛为 8 个固定挡位 `0.1x / 0.25x / 0.5x / 0.75x / 1.0x / 1.5x / 2.0x / 4.0x`，并把展开面板改成离散 segmented preset，明确区分“慢放观察 bit”与“快速浏览”两种使用方式。
* 按“主入口 -> 自动化索引 -> 专题文档”重组 Android agent 文档入口，并扩展 Flash adb/debug scenario 与 `capture-flash` wrapper，支持 `display`、`visual`、`playback speed`、`visual perf overlay` 等参数，便于按 style 批量抓取 `Mix + Pulse` 对齐数据。

### 修复 (Fixed)
* 修复 Flash `Visual / 8 bits / token hex-bin` 在 `Mix` 观察链路中的多处 bit 对齐问题，包括统一 `Pulse / Pitch / Lanes` 播放线位置、让 token 当前 byte 的 `hex/bin` 与 visual 使用同一 bit 口径，并补充 adb trace 与 debug overlay 来验证各 style 在 `1.0x` 和 `0.1x` 下的对齐结果。
* 修复默认 Flash style 命名在 Android 侧与库层 contract 长期分叉的问题，将 UI、XML 文案、样本文本和自动化输入统一从 `Steady` 改为 `Standard`；这次改名的原因是 `steady` 在英文里容易带出“长期且关系稳定的男女朋友”语义，与项目的科技风/科幻风格不匹配，而 `Standard` 更符合“标准说话风格”的产品表达，也能避免 Android 已写 `Standard`、libs 仍保留 `Steady` 导致后续开发混淆。
