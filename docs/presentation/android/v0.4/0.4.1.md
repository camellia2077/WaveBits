## [v0.4.1] - 2026-04-30

### 新增功能 (Added)
* 新增已保存音频的文件大小展示：`mini player` 底部 info 弹窗和播放器详情页现在会显示实际 WAV 文件大小，并与生成时间、输入来源、采样率和 payload 字节数一起帮助用户判断保存文件的体积来源。

### 技术改进/重构 (Changed/Refactor)
* 优化播放器播放进度同步策略，对 UI 进度刷新做节流，降低播放期间 `Visual`、详情页和底部控制区因高频 sample position 更新产生的重组压力。
* 优化 waveform 绘制前的数据准备，将 PCM 波形 bucket 预计算并量化，避免播放动画帧反复扫描完整采样数组，长音频和预览波形下的 `Visual` 页面滚动与播放更稳定。
* 优化 Flash / Pro 可视化预分析缓存：Flash 的 follow timeline bucket 与 PCM energy bucket 会复用小型缓存，Pro 解释视图也缓存已推导的展示状态，减少播放位置重复进入相近窗口时的频谱/状态分析成本。
* 优化已保存音频列表 metadata 读取，在 `MediaStoreSavedAudioLibraryGateway` 中缓存可读和缺失的 WAV metadata header，并在删除、重命名或 MediaStore row 变化时失效，避免反复进入 Saved 列表时重复打开每个 WAV 文件。
* 重构 `mini player` info 弹窗的数据组装，将详情行 model 与静态文案收口为更稳定的 remembered model，减少播放进度刷新期间的列表和 row model 分配。

### 修复 (Fixed)
* 修复当前 Android 全量测试和 ktlint 阻塞，更新资源一致性、品牌主题颜色解析、样例输入断言、codec action 期望值与开源许可界面格式，使 `:app:testDebugUnitTest`、`ktlintCheck` 和 `assembleStaging` 能在发布前通过。
