## [v0.2.1] - 2026-03-16

### 新增功能 (Added)
* 新增全局常驻迷你播放器条，固定显示在 Android app 底部 tab bar 上方；只要当前存在可播放的生成音频或已选中的库音频，就会在 `Audio`、`Library` 与 `Config` 三个 tab 中持续可见。
* 新增全局播放器详情 `Bottom Sheet`，点击迷你播放器条后可查看近场可视化动画、进度条、播放控制、播放模式，以及当前音频的摘要信息。
* 新增生成态与库音频统一的迷你播放器展示模型，在壳层级根据当前播放源自动切换标题、副标题和时长信息：生成态优先展示 mode 与 `flash` voicing 风格，库音频优先展示文件名、mode 与时长。
* 新增播放器详情中的生成态解析内容区；当播放源来自当前 mode 已生成音频时，可直接在详情层查看解析文本。
* 新增播放器详情中的库音频信息区；当播放源来自已保存音频时，可在详情层查看 mode、时长与保存时间。
* 新增 `Audio` 页编码进度区的阶段文案展示，在现有进度条上方直接显示 `Preparing input / Rendering PCM / Post-processing / Finalizing` 的当前阶段，不再只暴露百分比。
* 新增 `Audio` 页输入区的骰子样例按钮，可按当前 mode 随机替换为另一条示例文本，帮助用户在不手输内容的情况下快速体验 `flash / pro / ultra` 的编码效果。

### 技术改进/重构 (Changed/Refactor)
* 重构 `apps/audio_android` 的播放器布局层级，将播放能力从 `Audio` 页局部卡片提升为 app shell 级能力，形成“主内容区 + 常驻迷你播放器条 + 底部 tab bar”的整体结构。
* 重构 Android app 壳层，由 `AudioAndroidApp` 统一托管全局迷你播放器条、播放器详情 `Bottom Sheet` 与已保存音频列表 `Bottom Sheet`，不再由 `AudioTabScreen` 单独持有 tab 内播放器弹层。
* 重构 `Audio` 页职责，移除页内播放卡片，将页面重新收敛为“mode 切换 + 输入文本 + flash 发声风格 + 编码/解析 + 结果文本”的生成工作台。
* 重构播放器入口与列表入口的所有权，将已保存音频列表从 `Audio` 页内的局部入口提升为全局入口，可由迷你播放器条和播放器详情层直接打开。
* 新增壳层级 UI 状态与派生模型，用于统一表达“是否显示全局播放器详情层”“当前迷你播放器展示内容”“当前播放源对应的解析文本或库音频信息”等播放展示态。
* 删除旧的 `AudioPlaybackCard` 与旧 header 壳组件，统一改由全局迷你播放器条和播放器详情层承载播放交互。
* 重构 `NativeAudioCodecGateway` 与 `AudioSessionCodecActions` 的编码状态链路，将 one-shot `ShortArray` 结果收口为显式 `Success / Cancelled / Failed(code)`，并把 native `phase` 真正抬到 Android session state。
* 重构 `Audio` 页编码状态更新策略，保留 `encodeProgress` 的实时刷新，同时将 `statusText` 改为只在 phase 切换时更新，避免长文本编码期间每个轮询周期都重建状态文案。
* 重构 `Audio` 页样例输入链路，将单条 `exampleText` 升级为带稳定 `sampleId` 的多语言样例池，并让语言切换只映射当前样例文本，不再自动重新随机。

### 修复 (Fixed)
* 修复播放状态只能在 `Audio` 页可见的问题；现在切换到 `Config` 或 `Library` 时，当前播放状态和播放入口不会从界面中消失。
* 修复 `Audio` 页被播放卡片长期占据首屏空间的问题；现在输入区、模式切换和结果区拥有更稳定的主工作区，不再被播放器组件挤压。
* 修复已保存音频列表与播放器详情可能分散在不同页面层级的问题；现在两类底部弹层统一由 app shell 管理，并保持互斥，避免出现多个底部层同时叠加的交互冲突。
* 修复库音频选择会强制把用户带回 `Audio` 页的问题；现在从全局已保存音频列表选择库音频后，会保留当前 tab 语境，并继续通过全局播放器承接播放。
* 修复编码失败与取消共用“空 PCM”语义导致的上层误判；现在取消、失败和成功结果在 Android UI 状态中明确分流，不再把取消落成内部错误。
* 修复 `Audio` 页长文本编码时进度台阶过粗的问题；native 侧现在以更细的 sample 步长上报真实工作量进度，进度条与百分比更新更平滑。
* 修复 `Audio` 页示例文本与输入操作语义分离的问题；骰子按钮现在贴近输入框上方显示，并在 `flash / ultra` 模式下默认使用 `红袍守夜人正在点亮旧星图` 及其对应英日文本，不再保留旧的 `Hello Android / 你好，安卓 / こんにちは、アンドロイド` 默认示例。
