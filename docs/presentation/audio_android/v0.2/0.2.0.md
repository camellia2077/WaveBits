## [v0.2.0] - 2026-03-15

### 新增功能 (Added)
* 新增 Android XML 多语言资源 `apps/audio_android/app/src/main/res/values/strings.xml`、`apps/audio_android/app/src/main/res/values-zh/strings.xml` 与 `apps/audio_android/app/src/main/res/values-ja/strings.xml`，让 `Audio`、`Config`、`About`、`Licenses` 与应用标题支持英文、中文和日语。
* 新增 Config 页应用内语言切换，支持 `跟随系统`、`中文`、`English` 与 `日本語`，并通过 `AppCompatDelegate` 自动保留上次语言选择。
* 新增按语言切换的示例输入文本，英文默认示例为 `Hello Android`，中文下 `flash` 与 `ultra` 默认示例为 `你好，安卓`，日语下 `flash` 与 `ultra` 默认示例为 `こんにちは、アンドロイド`，`pro` 继续保持 ASCII 示例。
* 新增 `Preferences DataStore` 持久化 `selectedPalette`，重新打开 `apps/audio_android` 后继续使用上次选中的配色方案。
* 新增独立共享库 `libs/audio_runtime`，沉淀平台无关的播放会话状态机、seek 语义、样本位置换算与时间计算能力，为后续多表现层复用做准备。
* 新增 `libs/audio_io` 的 bytes-based WAV 编解码能力，并通过 Android 专用 `audio_io` native package / JNI bridge 暴露给 Kotlin，作为 Android 保存音频与读取已保存音频的统一 `wav <-> pcm` 边界。
* 新增 Audio 页播放卡片下载按钮，可将当前 mode 已生成的音频导出为 `wav` 并保存到 Android `Music/WaveBits/` 目录。
* 新增 Audio 页播放卡片已保存音频列表按钮，支持从底部弹出 `ModalBottomSheet` 浏览已保存音频，并选择条目加载为当前播放源。
* 新增已保存音频 mode 筛选：Audio 页底部列表支持 `全部 / flash / pro / ultra` 四档筛选，并默认优先落在当前 mode；`Library` 页也支持同口径筛选，便于在不做物理分库的前提下快速聚焦目标音频。
* 新增 Audio 页进度卡片播放模式控制，支持 `随机播放`、`单曲循环` 与 `列表循环` 三种续播策略；保存音频来源播放完成后会按所选模式自动切到下一首或重播当前音频。
* 新增 Audio 页播放控制区 `上一曲 / 播放暂停 / 下一曲` 三键布局；手动切歌会沿已保存音频列表顺序切换，并在切到目标曲目后立即开始播放。
* 新增底部 `Library` tab，集中展示所有已保存音频，并支持点击条目切回 Audio 页加载为当前播放源。
* 新增已保存音频管理能力，支持在 `Library` 页对音频执行分享、重命名和删除操作。
* 新增 `Library` 页长按进入多选模式，支持全选、批量删除和退出多选，便于一次性清理已保存音频。

### 技术改进/重构 (Changed/Refactor)
* 升级 `apps/audio_android/app/build.gradle.kts` 中的 `versionName`，将 Android 表现层版本从 `0.1.1` 更新为 `0.2.0`。
* 重构 Audio 页状态模型，将 `flash`、`pro`、`ultra` 的输入文本、生成 PCM、解析结果、状态文本和播放进度下沉到各自独立 session。
* 重构 Android 本地化链路，统一由 Android app 层消费 `UiText` 与 XML 资源，不再直接展示 Native 返回的英文消息字符串。
* 重构 JNI 与 Gateway 校验接口，将校验结果从 message string 改为错误码返回，并在 Android 侧统一映射为本地化文案。
* 继续轻拆 `apps/audio_android/app/src/main/java/com/bag/audioandroid/ui/AudioAndroidViewModel.kt` 职责，新增 `AudioSessionStateStore` 与 `PlaybackSessionReducer`，把 session 更新和播放状态归并从 ViewModel 编排层中抽离，降低播放控制、状态归并与界面状态写入的耦合。
* 轻拆 `apps/audio_android/app/src/main/java/com/bag/audioandroid/ui/screen/AudioTabScreen.kt`，把 mode 切换、输入操作、解析结果、播放控制四块 UI 拆到独立 composable，收窄顶层 screen 的职责边界，便于后续继续调整 Audio 页布局。
* 继续收瘦 `apps/audio_android/app/src/main/java/com/bag/audioandroid/ui/AudioAndroidViewModel.kt`，将应用壳层、输入/编解码、播放交互、已保存音频库四组动作分别下沉到 `AudioAndroidChromeActions`、`AudioAndroidSessionActions`、`AudioAndroidPlaybackActions` 与 `AudioAndroidLibraryActions`，让 ViewModel 回到“状态汇总 + 回调分发”的薄壳角色。
* 继续拆分 Audio / Library 页面文件：将原先集中在 `AudioTabSections.kt` 的 mode 切换、输入/结果卡片、播放卡片、saved audio sheet 与时间格式化工具拆到独立文件；将原先集中在 `LibraryTabScreen.kt` 的选择工具栏、列表项与对话框拆到独立文件，降低后续继续扩展页面时的单文件复杂度。
* 新增独立导出边界 `AudioExportGateway` 与 `MediaStoreAudioExportGateway`，将音频保存职责与编解码、播放 runtime 分离，并统一封装 `wav` 写入与 `MediaStore` 落盘逻辑。
* 新增 `SavedAudioLibraryGateway` 与 `MediaStoreSavedAudioLibraryGateway`，统一封装已保存音频查询、`wav` 读取与播放前加载逻辑，并在 Android UI 层引入 `AudioPlaybackSource` 区分“当前生成音频”和“已保存音频”两类播放源。
* 扩展 `SavedAudioLibraryGateway`，补充删除与重命名边界，并将底部列表与 `Library` 页统一收口到同一份已保存音频数据源。
* 重构 Android 播放链路，将 `Float progress` 升级为基于 `playedSamples / totalSamples / sampleRateHz` 的稳定播放状态，并通过独立 `playback_runtime_jni.cpp` 与 `PlaybackRuntimeGateway` 接入 `audio_runtime`。
* 继续收口 Android 数据层职责：新增 `SavedAudioRepository`、`AndroidIntentAudioShareGateway` 与 `PlaybackSourceCoordinator`，把导出、列表读取、分享和播放源解析从 `AudioAndroidViewModel` 中进一步下沉到专用协作层。
* 重构 `MediaStoreAudioExportGateway` 与 `MediaStoreSavedAudioLibraryGateway`，让它们只负责 `MediaStore` 插入/查询/重命名/删除和字节流读写，不再在 Kotlin 层手写 WAV header 或手动解析 WAV 字节。
* 重构 Audio 页进度区为音乐播放器式布局：进度条整行铺满，当前时间与总时长放到进度条下一行左右对齐，中间使用可拖动 Slider 统一承接 seek 交互，播放/暂停图标放到进度条区域下方单独居中显示。
* 重构 Audio 页整体布局为可上下滚动，并将播放卡片上移到输入区和解析结果区之前，减少长文本场景下播放状态与进度控制被挤出首屏的问题。
* 重构 Audio 页 mode 选择入口，将 `flash / pro / ultra` 从 radio 组改为页面顶部的横向 Material segmented 单选条，更贴近高频模式切换场景。
* 重构 Audio 页内容分区，将解析结果从播放控制卡片中拆出，改为独立的结果卡片显示，更贴近“播放区 + 歌词/文本区”的使用逻辑。
* 重构 Audio 页输入操作区，将输入框、`文本转音频`、`解析生成音频` 与 `清空输入文本` 收口到同一张卡片内，使编辑与动作入口保持同一语义分区。
* 微调 Audio 页操作分布，将 `解析生成音频` 与 `清空输入文本` 从输入卡片迁到解析结果卡片，使输入卡片更聚焦“编辑 + 生成”，解析卡片更聚焦“输出 + 后续处理”。
* 微调解析结果卡片操作区，将 `解析音频` 与 `清空输入` 改为同一行双按钮，并同步收短中文、英文、日文按钮文案，减少操作区纵向占高。
* 重构 Audio 页长文本交互，为输入区和解析结果区新增折叠/展开按钮，并为多行内容增加最大显示高度与区域内滚动，避免输入文本或解析结果无限撑高卡片。
* 重构 Config 页配色选择区，改为横向滚动的纯预览色板；单项不再显示标题/副标题文本，选中态通过 Material 默认 `check` 图标与外环表达，色板下方仅显示当前选中的配色名，并扩充为按彩虹顺序排列的高饱和主题色。
* 重构 Config 页配色选择区布局，将纯预览色板改为可展开/收起的多行排列，并通过 Material 默认箭头图标控制行折叠状态。

### 修复 (Fixed)
* 修复切换 mode 后继续展示上一模式 `generatedPcm`、解析结果和播放状态的串台问题。
* 修复播放和解析误用其他模式已生成 PCM 的问题，并在当前模式没有音频时统一提示 `该模式下还没有生成音频`。
* 修复切换 mode、重新生成或清空时旧播放未停止导致的叠播问题。
* 修复当前生成音频只能停留在应用内、无法按用户习惯保存到设备的问题；现在会优先取输入内容前 `12` 个可见字符作为文件名前缀，并按 `<preview>_<mode>_<yyyyMMdd_HHmmss>.wav` 导出，重名时自动追加 `_2`、`_3` 避让；输入为空时仍回退到 `wavebits_<mode>_<yyyyMMdd_HHmmss>.wav`。
* 修复播放卡片只能消费当前 session 生成音频的问题；现在可从已保存音频列表选择历史音频进行播放，且不会覆盖输入文本和解析结果。
* 修复已保存音频缺少集中浏览入口的问题；现在可在 `Library` 页统一查看全部已保存音频，并在删除、重命名后同步刷新到底部列表和库页。
* 修复已保存音频列表只能平铺全部内容的问题；现在 Audio 页底部列表和 `Library` 页都可按 mode 快速筛选，且 `Library` 页切换筛选时会自动退出多选，避免隐藏项仍留在选区里。
* 修复主题色切换后音频播放进度条与底部 Tab 背景未同步更新的问题，使其跟随当前选中的配色方案显示。
* 修复 `flash / pro / ultra` mode 切换条与进度条未明显跟随当前主题色的问题，改为显式使用当前主题的 `primary / primaryContainer` 配色。
* 修复版本信息、状态提示与页面可见文案中的硬编码英文问题，改为跟随应用语言或系统语言显示。
* 修复音频播放区只有单向百分比进度、无法按用户熟悉的播放器心智做暂停/继续/拖动定位的问题；现在拖动时会暂停，松手后 seek 到新位置，并按拖动前状态决定是否恢复播放。
* 修复解析结果过长时持续挤占页面空间的问题，并为解析结果卡片新增垃圾桶按钮，可单独清空输出内容而不影响输入文本和已生成的音频。
