## [v0.6.1] - 2026-05-03

### 新增功能 (Added)
* 新增 Flash `Zeal` / `Void` 正式库层风格，扩展 `bag_flash_signal_profile`、`bag_flash_voicing_flavor`、core config 和 WAV metadata 枚举，让 Android、CLI 和库级测试可以使用同一套公开 contract 生成、保存和恢复新增情绪音色。
* 新增 `bag_describe_flash_signal(...)`，从 core 真实 signal profile 和 payload layout 派生 low/high carrier、bit duration、payload silence 和 decode path 信息，供 Android 展示生成参数时复用库层事实而不是在前端重复维护规则。
* 新增 WAV header probe 能力，`audio_io` 现在可通过 `ProbeMonoPcm16Wav(...)` / `audio_io_probe_mono_pcm16_wav(...)` 读取 sample rate、channel、bit depth、PCM sample count、data byte count、file byte count 和 duration，支撑保存音频和未保存生成结果复用同一套音频信息。

### 技术改进/重构 (Changed/Refactor)
* 调整 Flash 风格 timing 和音色底层参数：`Steady` 改为略快的 `15/16 frame` timing，`Hostile` 改为 `7/8 frame` timing 并强化更硬 attack、短 release 和 byte boundary click，`Void` 改为 `220 / 440 Hz`、`2.5x frame` timing 和更长 release，`Zeal` 改为可解码的变长 bit window、变 carrier 和标点 gap-aware payload cadence。
* 重构 Flash payload layout 渲染，从固定 low/high bit template 改为按 chunk sample count 和 carrier 逐段渲染，使 `Zeal` 这类变速、变频、带 punctuation pause 的 payload 仍能发布准确 follow timeline 并走专用 decode。
* 拆分 `audio_io_api.cpp`、`bag_api_impl.inc`、`bag_api_result_marshalling_impl.inc`、`bag_api_encode_entrypoints_impl.inc` 和 `transport/follow_impl.inc`，把 C ABI entrypoint、metadata marshalling、result lifecycle、signal info、encode job、text tokenization、lyric line 与 payload timeline 等职责分开，降低后续跨 Android / CLI 边界改动的文件级耦合。
* 更新 `libs/AGENTS.md` 和 `libs/README.md` 为任务导向的文档入口说明，避免把所有 libs 文档作为改动前的强制阅读清单。

### 修复 (Fixed)
* 修复 Flash follow 和 decode 回归覆盖不足的问题，新增 `Zeal` / `Void` roundtrip、wrong-config decode、variable cadence length、signal info 派生和 WAV metadata roundtrip 测试，确保新增/调整后的风格仍以可稳定 decode 为优先级。
* 修复 WAV 信息读取只依赖完整 decode 的限制，header probe 现在可直接从 WAV bytes 获取大小和时长信息，并覆盖 metadata WAV 的 C ABI probe roundtrip。
