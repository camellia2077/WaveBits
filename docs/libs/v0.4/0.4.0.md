## [v0.4.0] - 2026-03-15

### 新增功能 (Added)
* 新增 `bag.flash.signal`，对外提供 `BfskConfig`、payload layout 和 clean `byte <-> PCM` encode/decode 能力。
* 新增 `bag.flash.voicing`，对外提供 `FlashVoicingConfig`、`FlashVoicingDescriptor`、`FlashVoicingResult` 与 `ApplyVoicingToPayload(...)` 的最小 no-op 风格层骨架。

### 技术改进/重构 (Changed/Refactor)
* 重构 `bag.flash.phy_clean`，把原先混在 facade 里的 clean BFSK 逻辑下沉到 `bag.flash.signal`，并让 `phy_clean` 收口为 text facade + decoder 组合。
* 重构 `bag.flash.phy_clean::EncodeTextToPcm16()` 的正式 `flash` 输出链路，改为显式走 `text -> bytes -> payload layout -> clean signal -> voicing`，同时加入“当前 voicing 必须保持 clean decode 兼容长度”的保护。
* 重构 encode job 的进度口径，让 `flash`、`pro` 与 `ultra` 不再使用固定阶段配额；新的进度统一按 `completed_work / total_work` 累计，并继续通过既有 `EncodeProgressPhase` 对外上报。
* 重构 `bag.flash.signal`、`bag.flash.voicing`、`bag.pro.phy_clean` 与 `bag.ultra.phy_clean` 的进度推进方式，将 payload 渲染与后处理从按 `chunk` / `symbol` 计步改为按已完成 sample 数计步，并加入固定 sample 步长节流，减少长文本编码时的进度台阶感。
* 重构 `flash` formal voicing 的 postprocess 进度预算，把 `preamble`、payload voicing、`epilogue` 与 `spectral texture` 全部并入同一真实后处理工作量，不再把非 payload 区域压缩进尾部少量固定进度。
* 重构 `bag.flash.voicing`，加入 payload-length-preserving 的安全风格元素：短 attack / release 包络、低增益二次 / 三次谐波，以及按 byte 边界注入的极短 click，并通过 gain budget 限制叠加强度。
* 继续增强 `bag.flash.voicing` 的非编码节奏骨架，让默认 preamble / epilogue 具备更明确的 `16` 拍 grouped burst / `4x4` 分组感，并为 payload 内补充 nibble 级轻量 accent，同时保持 byte 边界 accent 更强、decode 仍只依赖 clean payload。
* 继续增强 `bag.flash.voicing` 的非编码频谱质感，为默认 formal `flash` 输出加入轻度带通电话感、低增益确定性噪声底与轻微软削波，并限定这些频谱后处理只在真正存在 payload voicing 的场景启用，不污染 `no-op` 与 trim-only 基线。
* 继续增强 `bag.flash.voicing` 的多层音色，在 payload 内部加入轻量金属层、低电平空气层与固定频率微弱 AM 脉动，增加机械厚度，同时保持这些副层不承载新信息、不改 preamble / epilogue 语义，并维持 `click-only`、`trim-only` 与 `no-op` 基线稳定。
* 继续增强 `bag.flash.voicing` 的仪式化外观，为默认 preamble 加入非语义化的唤起 sweep 与确认脉冲，为 epilogue 加入闭锁脉冲与收束尾部，让开始 / 结束更易被听感区分，同时仍不引入固定文本内容或 payload 语义变化。
* 重构 `bag.flash.voicing` 的内部风格边界，新增 `FlashVoicingStyle` 与 `ApplyVoicingToPayloadWithStyle(...)`，把当前正式默认听感固化为 `coded_burst`，同时新增 decode 结果完全一致但听感更偏连续共鸣的 `ritual_chant` 路径；随后又将这组 style 统一公开到 `CoreConfig.flash_style` 与 `bag_api.h`，不再只停留在 core 内部 helper。
* 继续细化 `ritual_chant` 的 payload 包络，让其在不改 decode 的前提下使用更长 release、更高 payload envelope floor，并减弱 bit 与 bit 之间的断开感；同时进一步压低边界 click、抬高 resonance，让听感更接近连续吟唱而不是逐段报码。
* 重构 formal `flash` 的内部 trim / decode helper，使其可以按 `FlashVoicingStyle` 感知前后壳长度；`coded_burst` 继续保持原有时长，而 `ritual_chant` 改为显著更长的 preamble / epilogue，并通过 style-aware trim 保持同版本下的 decode 自洽，同时让整秒级 UI 时长显示也能直接看出差异。
* 继续细化 `ritual_chant` 的固定壳层外观，只在非 payload 区域加入更明确的分句停顿：preamble 改为三段式短停顿，epilogue 改为两段式短停顿；这些停顿只改变外壳听感和总时长，不改变 payload timing、bit 映射或 decode 结果。
* 重构 `bag.flash.signal` 的 timing 边界，将 payload 时长权威口径从 `bit_duration_sec` 改成稳定的 `samples_per_bit`，并让 `ritual_chant` 通过 `3x` 的每-bit 样本数真正变慢；同一份 `flash_style` 配置会同时驱动 signal timing、voicing、trim 与 decode。
* 新增 `bag.flash.voicing` 的 trim descriptor / payload trim 纯函数，并让 `bag.flash.phy_clean::DecodePcm16ToText()` 先 trim 非 payload 区域，再把 payload PCM 交给 `bag.flash.signal` 解调，为后续 preamble / epilogue 铺路。
* 新增固定 preamble / epilogue，并把 formal `flash` 默认输出长度升级为 `preamble + payload + epilogue`；同时更新 `artifact_tests` 与 `leaf_module_smoke` 的长度断言口径，明确区分 clean helper 和正式 `flash` 输出。
* 重构 `bag_api.h`、CLI 与 Android 接线，让 `flash_style` 回到统一稳定配置面，不再通过 Android 私有 JNI 分叉 encode/decode；`coded_burst` 继续作为默认 style 和零值语义。
* 新增独立的 `flash_voicing_tests`，把 `bag.flash.voicing` 的 no-op、envelope、harmonic、click 与确定性回归从 `modules_leaf_smoke` 中进一步拆成可独立定位的测试面。
* 为 `flash_voicing_tests` 增加默认 `coded_burst` 对齐验证，以及 `coded_burst` / `ritual_chant` “PCM 不同但 decode 结果一致”的回归覆盖，正式把“不同听感、相同 payload 语义”钉进测试。
* 为 `flash_voicing_tests` 增加 `ritual_chant` tail energy 回归，固定“更长 release / 更弱 bit 断开感”的调音方向，避免后续回调时丢失连续吟唱特征。
* 为 `flash_voicing_tests` 增加 formal `flash` style-aware 回归，覆盖 `ritual_chant` 长于 `coded_burst`、`ritual_chant` style-aware trim 后仍可 roundtrip，以及默认 formal flash 继续对齐 `coded_burst` 基线。
* 重构 `apps/audio_android/native_package` 的 `flash` 私有包装层，新增与 `bag.flash.signal` 对齐的 `android_bag/flash/signal.h` 和 `audio_core_flash_signal.cpp`，保持 Android lane 的 package-owned 结构同步。
* 重构 `Test/modules/leaf_module_smoke.cpp` 与相关静态 policy，新增 `bag.flash.signal`、`bag.flash.voicing` 的直接覆盖，同时保留 `bag.flash.phy_clean` 的 text facade roundtrip gate。
* 为 `api_tests`、`artifact_tests` 与 `modules_leaf_smoke` 增加 `flash_style` 回归，覆盖 `ritual_chant` 总长度增长、匹配 style 可 roundtrip、错 style 不应被当成成功 roundtrip，以及非法 `flash_style` 的显式拒绝语义。
* 为 `api_tests` 增加 encode job 进度回归，固定“进度单调不减、phase 不回退、`flash` 会进入 `Postprocessing`、`pro/ultra` 不会误报 `Postprocessing`、长文本会产生多个中间进度点”的运行时约束。
