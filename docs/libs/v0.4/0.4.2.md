## [v0.4.2] - 2026-04-22

### 技术改进/重构 (Changed/Refactor)
* 统一 `libs` 侧品牌与版本生成口径，把 `wavebits` 前缀的构建辅助与 core version 宏切换为 `flipbits`：`libs/audio_api`、`libs/audio_io`、`libs/audio_runtime` 的 host test 注册现统一改为 `flipbits_add_host_test(...)`，`libs/audio_core` 生成头与 `CoreVersion()` 也改为消费 `FLIPBITS_CORE_VERSION_TEXT` / `FLIPBITS_CORE_GENERATED_INCLUDE_DIR`。
* 重构 `audio_io` 的 metadata 命名边界，将 `wav_io.h`、backend、bytes impl 与 C ABI 中的 `WaveBitsAudioMetadata*` 类型、枚举、解析函数和状态消息统一改名为 `FlipBitsAudioMetadata*` / `ParseFlipBitsAudioMetadata(...)`，避免库侧 contract 继续暴露旧品牌前缀。
* 重构 `audio_io` 的 `WBAG` metadata bytes 实现，把 version、mode、flash style、`created_at`、`frame_samples`、`pcm_sample_count`、`app/core version` 的 chunk offset 与 chunk size 显式抽成常量，减少后续继续演进 metadata payload 时的魔法数字扩散。

### 修复 (Fixed)
* 修复 `audio_io` metadata 回归覆盖的可维护性问题：`libs/audio_io/tests/unit_tests.cpp` 现改为使用显式的 metadata payload offset 常量校验损坏 version / mode / flash style / timestamp / version text 等异常路径，降低 `WBAG` 布局调整后测试一起失真的风险。
* 修复 `audio_io` C ABI 与库内实现的命名漂移，把 `audio_io_metadata_status_message(...)` 等对外状态文案同步切到 `FlipBits metadata` 口径，避免 parse 成功、失败与 not found 路径继续输出旧的 `WaveBits metadata` 说明。
