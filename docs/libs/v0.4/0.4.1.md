## [v0.4.1] - 2026-04-19

### 新增功能 (Added)
* 新增正式 `audio_io` C ABI：`audio_io_api.h` 现统一提供 `mono PCM16 WAV` encode/decode、完整 `WBAG` metadata enum/view/result、owned string 释放与“一次调用返回完整 decode 结果”的跨语言边界，作为 CLI 与 Android 共用的多前端 native contract。
* 新增库内测试入口：`libs/audio_api/tests`、`libs/audio_io/tests`、`libs/audio_runtime/tests` 现成为各自正式测试落点，并把 `runtime_tests`、`api_tests`、`unit_tests` 的 target 注册下沉到库侧 `CMakeLists.txt`。

### 技术改进/重构 (Changed/Refactor)
* 重构 `audio_io` 的 native 边界与实现组织，让 header consumers、module importers 与新的 C ABI 共享同一套 `wav_io` backend；`audio_io` 现在同时承载 `wav_io.h`、`audio_io.wav` module 与 `audio_io_api.h` 的统一实现，不再为不同前端维护分叉协议。
* 重构 `bag_api` 测试面，把原先单文件 `api_tests` 按职责拆成 `sync` / `async` / `flash` + shared support，多文件继续保留同一个 `api_tests` 测试名，降低后续继续补 contract case 时的文件膨胀与定位成本。
* 重构 `bag.flash.voicing` 内部实现边界，将 `voicing_impl.inc` 拆成 support / texture / shell 三层内部 include，在不改变外部行为的前提下收窄 `audio_core` 中 formal `flash` 调音与壳层演进的维护面。

### 修复 (Fixed)
* 补齐 `bag_api` 的 decoder / poll 契约回归，锁定分块 `PCM push`、`poll_result` 缓冲边界、重复 `poll` / `reset`、以及 `flash / pro / ultra` 在真实 chunked 输入下的稳定 decode 语义，减少后续实现漂移。
* 补齐 `audio_io` 的 metadata 异常路径回归，固定损坏 `WBAG` chunk、metadata 截断、未知 `mode` / `flash style`、非法版本与 canonical WAV 无 metadata 时的解析状态表达，避免 WAV 成功与 metadata 失败语义再次耦合。
* 补齐 `audio_runtime` 的状态机与进度回归，覆盖 `stopped`、`failed`、`scrub_canceled`、`elapsed_ms` / `total_ms` / `progress_fraction`、越界 scrub 与 completed 后收敛行为，锁定 CLI / Android 共用 runtime 逻辑的纯逻辑边界。
