## [v0.3.3] - 2026-03-14

### 新增功能 (Added)
* 新增 `python tools/run.py clean`，统一清理默认 host 构建目录，并支持按 `host`、`android`、`artifacts`、`python` scope 分层执行清理。
* 新增 `docs/libs/v0.3/v0.3.3.md`，按当前发布历史模板收口本轮 `audio_core` 版本更新。

### 技术改进/重构 (Changed/Refactor)
* 重构 host-side `import std;` policy 与 current-state/docs 口径，将 `host_import_std` 从 audited pilot 改写为 required baseline，并固定 `14` 个 `audio_core` promoted module interfaces 的 `import-std-only` 终态。
* 重构 root host 支持策略，固定为单一 `clang++ + Ninja + build/dev` 主线，并让 root host 无条件要求当前 `import std;` baseline。
* 重构 `python tools/run.py` 的帮助输出为分层结构，根命令只保留主命令概览，详细参数下沉到 `python tools/run.py <command> --help`。
* 归档已完成的 `temp/audio-core-module-first-and-import-std-control-tower.md`、`temp/audio-core-current-deviations-closeout-plan.md`、`temp/import-std-pilot-promotion-gate.md` 与 `temp/import-std-required-baseline-feasibility.md` 到 `temp/archive/`，结束这轮 host import-std 收口执行面。

### 弃用/删除 (Deprecated/Removed)
* 删除 root `CMakeLists.txt` 中的 `WAVEBITS_HOST_IMPORT_STD_PILOT` 与 `WAVEBITS_HOST_IMPORT_STD_PILOT_ENABLED` 命名层，结束 root host `import std;` pilot 开关。
