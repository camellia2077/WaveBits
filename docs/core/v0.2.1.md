## [v0.2.1] - 2026-03-12

### 新增功能 (Added)
* 新增 host named modules 试点与 modules on/off 双路径构建能力，为后续让 `libs` 代码逐步切到 modules 建立可回滚入口。
* 新增 `bag.common.config`、`bag.common.error_code`、`bag.common.types`、`bag.common.version`、`audio_io.wav`、`bag.flash.codec`、`bag.pro.codec`、`bag.ultra.codec`、`bag.transport.compat.frame_codec` 等叶子模块。
* 新增 `bag.transport.decoder`、`bag.flash.phy_clean`、`bag.fsk.codec`、`bag.pro.phy_clean`、`bag.ultra.phy_clean`、`bag.pro.phy_compat`、`bag.ultra.phy_compat` 等中层模块。
* 新增 `bag.transport.facade` 与 `bag.pipeline`，补齐 `libs` 在 host modules 路径下的汇聚层。
* 新增 `Test/modules/` 下的一组 module-first 验证入口，用于覆盖基础模块、叶子模块、中层模块、facade / pipeline 与内部消费面收口后的主路径验证。
* 新增 compatibility header inventory、consumer boundary、module topology、Android native strategy 等文档与对应静态门禁。

### 技术改进/重构 (Changed/Refactor)
* 重构 `transport` 依赖拓扑，从 `transport.h` 中拆出 `bag/transport/decoder.h` 契约头，目的是先收紧 `libs/audio_core` 的边界，为后续 modules 化减少循环依赖。
* 重构 `bag/pro/text_codec.h` 的跨 mode 公开依赖，移除对 `bag/ultra/codec.h` 的公开耦合，目的是让 `libs` 内部头与未来 module 边界按 mode 收口。
* 重构 `bag_api.cpp`，让 host modules 路径直接 `import bag.common.*` 与 `import bag.transport.facade`，同时保持 `bag_api.h` C ABI 不变，目的是让 `libs/audio_api` 的实现层开始真正消费 modules。
* 重构 host 根目录 `CMake` 与 `tools/run.py` 的默认构建口径，改为优先走 modules 路径并保留 `--no-modules` fallback，目的是把 `libs` 的 modules 化从实验路径推进为 host 默认路径。
* 重构 `audio_io` 的 module/header 共存策略，拆出共享实现头，稳定 `audio_io.wav` 与 `wav_io.h` 的双路径消费，目的是在不破坏头文件边界的前提下让 `libs/audio_io` 也进入 modules 迁移路线。
* 重构 compatibility layer 的分级与冻结规则，明确长期边界头、中期兼容头与历史过渡头的职责，目的是为后续继续让 `libs` 代码退出旧 include/bridge 路径做准备。
* 重构测试拓扑与验证边界：`artifact_tests` 纯化为 boundary-first，`unit_tests` 收回内部/兼容验证角色，并把 module-first 验证系统化，目的是让 `libs` 使用 modules 的主路径拥有独立且稳定的验证面。

### 修复 (Fixed)
* 修复 `audio_core` 公开头之间的跨层与跨 mode 依赖缠绕问题，降低后续 `.cppm` 模块化时的循环依赖风险。
* 修复 host 默认 modules 路径缺少稳定回退口径的问题，补齐 `--no-modules`、compatibility inventory 与边界规则。
* 修复 CLI / JNI / Android native 容易重新直接依赖内部 `bag/...` 头的问题，补齐 consumer-boundary 静态门禁。
* 修复 `libs/*/src` 与 `Test/*` 持续扩大对 compatibility headers 依赖面的风险，补齐 compatibility / test-boundary 静态检查。
* 修复 `artifact` 与内部实现测试混合依赖造成的验证语义不清问题，把 boundary-first 与 module-first 职责重新分层。
