## [v0.2.0] - 2026-03-08

### 新增功能 (Added)
* 新增 `pro` 模式的 clean `DTMF-like` 双音物理层，实现 `ASCII byte -> nibble symbol -> PCM` 的正式主链路。
* 新增 `ultra` 模式的 clean `16-FSK` 物理层，实现 `UTF-8 byte -> nibble symbol -> PCM` 的正式主链路。

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs/audio_core/src/transport/transport.cpp`，将 `pro` 与 `ultra` 的主链路统一切到各自的 clean PHY，不再走旧的 compat framed-over-FSK 入口。
* 重构 `libs/audio_core/include/bag/pro/` 与 `libs/audio_core/include/bag/ultra/`，明确分离信息层 codec 与 clean PHY 职责。
* 重构 `Test/` 分层用例与 `docs/testing.md`，统一按 `flash / pro / ultra` 的真实模式语义验证 PCM 长度、roundtrip 与失败口径。
* 升级 `libs/audio_core` 版本到 `0.2.0`，并通过 `bag_core_version()` 对外暴露。

### 修复 (Fixed)
* 修复 `pro` 仍沿用旧十进制 payload 与 compat frame 语义的问题，恢复为 ASCII-only 的正式 byte/nibble 模式。
* 修复 `ultra` 仍依赖旧 framed `FSK` 路径的问题，改为基于 UTF-8 byte 的 clean `16-FSK` 模式。
