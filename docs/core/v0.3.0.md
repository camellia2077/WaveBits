## [v0.3.0] - 2026-03-12

### 新增功能 (Added)
* 新增锁定 `bag_api` target 与 `bag_api.cpp` host-only `import std;` 接线的边界标准化门禁。
* 新增防止 `pro/text/frame` 与 `fsk` wrapper 回流的 compatibility-eviction policy。
* 新增锁定 `bag_api.cpp`、`unit_tests` 与 `libs/audio_core/src/*.cpp` 的 legacy fallback 继续走 `bag/legacy/**` 的 bridge-retirement policy。
* 新增并扩展 `libs/audio_core/include/bag/legacy/**`，作为 Android / no-modules / legacy fallback 的显式声明面。

### 技术改进/重构 (Changed/Refactor)
* 重构 `bag_api.cpp` 到更靠近长期边界的 host-only standardization，host 路径直接消费 modules 并在实现内部切到 `import std;`，目的是让 `libs/audio_api` 边界后的实现层继续朝 modules 路径收口。
* 重构 `unit_tests` 到 `bag.flash.phy_clean`、`bag.pro.codec`、`bag.transport.compat.frame_codec` 的直接覆盖，撤出对历史 wrapper headers 的依赖，目的是让 host 主验证面不再阻塞 `libs` 的 compatibility retirement。
* 重构 `libs/audio_core/src/*.cpp` 的 no-modules fallback，统一改走 `bag/legacy/**`，目的是把 host 主线与 legacy declaration surface 的职责拆开，让 `libs/audio_core` 主线代码真正退出 shared bridge headers。
* 重构 `link/*`、`phy/*` 这类预留接口头为双态入口：C++20 路径直接 `import bag.common.*`，legacy 路径走 `bag/legacy/common/*`，目的是在不破坏预留接口的前提下继续清空 `libs` 主树里的旧 bridge 依赖。
* 重构 compatibility inventory、repo-map、module topology、testing、tools 文档到删除后口径，明确 `audio_core` 主线 bridge / compatibility surface 已经收完，下一步可以把焦点转向 `audio_io.wav`。

### 修复 (Fixed)
* 修复 `bag_api.cpp`、`unit_tests` 与 `audio_core/src/*.cpp` 长期继续借用共享 bridge headers 的问题，避免 Android / legacy surface 持续绑架 host 主线。
* 修复 `audio_core` compatibility-only wrappers 仍作为主消费入口残留的问题，删除 `pro/text_codec.h/.cpp`、`pro/frame_codec.h/.cpp`、`fsk/fsk_codec.h`。
* 修复 `audio_core` 剩余 shared bridge headers 长期停留在主线的问题，删除 `common/*`、`flash/*`、`pro/*`、`transport/*`、`pipeline/*`、`ultra/*` 里承担 bridge 语义的那一整圈头文件。
* 修复 host 默认路径、legacy `--no-modules` 路径与 Android `C++17` 路径三者职责混淆的问题，让 `bag/legacy/**` 成为唯一的声明 carve-out surface。

### 弃用/删除 (Deprecated/Removed)
* 删除 `audio_core` 主线 shared bridge headers，结束 `include/bag/**/*.h` 继续承担 host 主路径 bridge 语义的历史状态。
* 删除 `audio_core` 主线对 compatibility-only wrappers 与 shared bridge headers 的正式依赖，把 modules-first 保持为 `libs` 代码的唯一默认方向。
