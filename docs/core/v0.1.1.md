## [v0.1.1] - 2026-03-08

### 新增功能 (Added)
* 新增 `bag_encode_text()` 与 `bag_free_pcm16_result()`，支持平台层通过稳定 C API 完成编码。
* 新增 `flash / pro / ultra` 三种传输模式的最小闭环，其中 `pro / ultra` 采用协议帧封装后复用现有 FSK 物理链路。

### 技术改进/重构 (Changed/Refactor)
* 重构共享代码目录，将 `apps/audio_core/core` 迁移到 `libs/audio_core`，将 `apps/audio_core/api` 迁移到 `libs/audio_api`，并将 `modules/audio_io` 迁移到 `libs/audio_io`。
* 重构构建链路，统一由根 `CMakeLists.txt` 装配 `bag_core`、`bag_api` 与 `audio_io`，`audio_cli` / `audio_android` 不再内联 `core/src`。
* 重构 `libs/audio_io`，删除重复的编码/解码实现，仅保留 `wav_io` 文件 I/O 边界。
* 固定 `pro / ultra` 的 MVP 长文本策略为“单帧上限，超长报错”，当前不做自动拆帧与多帧重组。

### 修复 (Fixed)
* 修复根 `CMakeLists.txt` 指向 `build/apps/audio_cli/cmake` 的错误目录。
* 修复 Android JNI 与 Windows CLI 直接依赖 `bag/...` C++ 头文件的问题，统一改为通过 `bag_api.h` 访问内核能力。

### 弃用/删除 (Deprecated/Removed)
* 删除旧路径 `apps/audio_core/` 与 `modules/audio_io/` 下的共享代码目录。
