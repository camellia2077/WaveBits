## [v0.3.1] - 2026-03-13

### 新增功能 (Added)
* 新增 `audio_io_boundary` 静态检查分组，锁定 `audio_io` 的双入口边界、private backend 拆分、`sndfile` 依赖收口和 retired shared headers 不回流。
* 新增 `libs/audio_io/src/wav_io_backend.h` 与 `libs/audio_io/src/wav_io_backend.cpp`，将 `WAV` 文件读写的私有 backend 从 shared implementation headers 中独立出来。
* 新增 `Test/include/test_audio_io.h`，并扩展 `Test/unit/unit_tests.cpp` 与 `Test/modules/leaf_module_smoke.cpp`，为 `wav_io.h` / `audio_io.wav` 双入口补齐正式 contract 测试，覆盖 mono roundtrip、sample rate 保持和缺失文件失败语义。
* 新增 `libs/audio_core/include/bag/internal/**` 的剩余 post-legacy internal declaration headers，覆盖 `flash/pro/ultra/pipeline/transport` 与 `common/version` 的 `C++17` 声明面。

### 技术改进/重构 (Changed/Refactor)
* 重构 `libs/audio_io` 的实现链路，改为 `libs/audio_io/include/wav_io.h` 与 `libs/audio_io/modules/audio_io/wav.cppm` 双入口继续共存、`libs/audio_io/src/wav_io.cpp` 与 `libs/audio_io/modules/audio_io/wav_impl.cpp` 作为薄 front-end、`libs/audio_io/src/wav_io_backend.cpp` 作为唯一 private backend owner。
* 重构 `libs/audio_io/CMakeLists.txt` 的 host modules 装配方式，为 `audio_io.wav` 的实现单元与 header 路径统一接入 host-only `import std;` 试点，同时保持 legacy `C++17` 路径不变。
* 重构 `docs/architecture/compatibility-layer-inventory.md`、`docs/architecture/module-topology.md`、`docs/architecture/repo-map.md`、`docs/architecture/android-native-strategy.md`、`docs/testing.md` 与 `tools/README.md`，冻结 `bag_api.h` 与 `wav_io.h` 的长期边界口径，明确 Android 继续走 `CMake 3.22.1 + C++17 + bag_api.h` 例外路线，并将 `--no-modules` 收口为 retirement baseline。
* 重构 `tools/wavebits_tools/commands/verify.py`、`tools/wavebits_tools/commands/module_structure_policy.py`、`tools/wavebits_tools/commands/boundary_policy.py`、`tools/wavebits_tools/commands/host_import_std_policy.py`、`tools/wavebits_tools/commands/audio_io_boundary_policy.py`、`tools/wavebits_tools/commands/compatibility_policy.py` 与 `tools/wavebits_tools/commands/retirement_policy.py` 的用户可见规则描述，让 `verify --list-checks` 直接表达当前长期规则与主线路径定义。
* 重构 `libs/audio_api/src/bag_api.cpp`、`Test/unit/unit_tests.cpp`、预留接口头与 `libs/audio_core/src/*.cpp` 的 `C++17` fallback，统一从 `bag/legacy/**` 切到 `bag/internal/**`，同时保持 host modules 主线与 Android `C++17` 例外路线不变。
* 重构 `docs/notes/legacy-retirement-preconditions.md`、`docs/notes/legacy-release-gates.md` 与 `docs/core.md` 的当前状态口径，把阻塞点从 `bag/legacy/**` direct owner 问题收口到 Android 间接依赖链与 `--no-modules` baseline 退出条件。

### 修复 (Fixed)
* 修复 `audio_io` module/header 双入口长期共用 shared-header implementation chain 的问题，避免 `sndfile`、`filesystem` 与标准库重依赖继续停留在共用头中。
* 修复 `audio_io.wav` 实现单元中 backend 声明错误附着到命名模块导致的链接不一致问题，改为在 global module fragment 引入 backend 声明并保持普通 TU ABI。
* 修复 `audio_io` host-only `import std;` 只停留在外层接口的问题，让 `wav.cppm`、`wav_impl.cpp` 与 `wav_io.cpp` 的 host 路径达到一致口径，同时避免把 `import std;` 误扩到 `sndfile` backend TU。
* 修复长期文档与 `verify --list-checks` 仍保留迁移阶段表述的问题，避免把已经收口的 host-side modules 主线误读为仍在持续扩面中的迁移工程。
* 修复 Android 与 `build/legacy-host --no-modules` 仍直接依赖 `bag/legacy/**` declarations 的问题，让剩余阻塞项只保留在 `C++17` internal implementation chain 与 gate/packaging 决策层面。
* 修复 `retirement` 静态门禁仍把 `bag_api.cpp` 与 `libs/audio_core/src/*.cpp` 视为批准 legacy owner 的问题，`verify` 现在会阻止任何 direct `bag/legacy/**` include 回流。

### 弃用/删除 (Deprecated/Removed)
* 删除 `libs/audio_io/src/wav_io_shared.h` 与 `libs/audio_io/src/wav_io_detail.h`，结束 `audio_io` 继续依赖 shared implementation headers 复用主实现的旧结构。
* 删除 `bag_api.cpp`、`unit_tests.cpp`、预留接口头与 `libs/audio_core/src/*.cpp` 对 `bag/legacy/**` 的 direct include 依赖，清空 `bag/legacy/**` 的 direct owner 集合。
