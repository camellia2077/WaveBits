## [v0.3.2] - 2026-03-13

### 新增功能 (Added)
* 新增 `temp/audio-core-module-first-and-import-std-control-tower.md`，把 `audio_core` 的总体目标、总体进度、长期边界与当前偏差统一收口到 module-first / host-side `import std;` 总控入口，并明确 `bag_api.h`、`wav_io.h`、Android `C++17` 例外路线与 host `--no-modules` 的长期定位。
* 新增 `temp/legacy-preconditions-burn-down-phase-3-and-4-no-modules-baseline-and-android-exit-criteria.md` 与 `temp/legacy-preconditions-burn-down-phase-4-no-modules-baseline-gate-reduction-and-exit-actions.md`，并将 Android native consumption split 的正式路线回写到 `docs/architecture/android-native-strategy.md`，把 Android 独立 gate 与 host-only `--no-modules` 退出动作拆成独立执行面。
* 新增 `temp/audio-core-current-deviations-closeout-plan.md`，把总控文档下的 5 个 `Current Deviations` 收口成单一 closeout plan，并固定收尾顺序为 Android 平台例外、host-only `--no-modules`、`bag/internal/**`、dormant `bag/legacy/**` 与 host-side `import std;` 的受审扩面。

### 技术改进/重构 (Changed/Refactor)
* 重构 `temp/legacy-preconditions-burn-down.md` 的执行主线，在 `Phase 1` 与 `Phase 2` 收口后把当前活动入口切到 `Phase 3 / 4`，并将默认顺序固定为“Android native consumption split 先完成最小落地，再推进 host `--no-modules` baseline 退出”。
* 重构 Android native 装配方式，app `CMake` 从直接 `add_subdirectory(libs/audio_core)` / `add_subdirectory(libs/audio_api)` 切到 `add_subdirectory(apps/audio_android/native_package)` 并只链接 `bag_android_native`，将 Android `C++17` internal chain 收口为平台专用 packaging lane。
* 重构 `libs/audio_core/src` 的模块分类口径，明确 `common/version`、`transport`、`flash/pro/ultra` 主实现统一按 modules-only internal implementation 收口，Android `C++17` 例外继续限制在 `apps/audio_android/native_package/` 私有包装层。
* 重构预留接口头的非-modules 声明边界，新增 `libs/audio_core/include/bag/interface/common/{config,error_code,types}.h`，并将 `link_layer.h`、`fun_phy.h`、`pro_phy.h` 从 `bag/internal/common/*` 切到这组 reserved interface declaration headers。
* 重构 `bag/interface/common/*` 的长期定位口径，明确这组头固定为 `link/*` / `phy/*` 预留接口头专用的长期非-modules 声明边界，不再作为待退休偏差或 `import std;` 扩面目标。
* 重构 `apps/audio_android/native_package/CMakeLists.txt` 的 target ownership，改为消费 `apps/audio_android/native_package/src/CMakeLists.txt` 暴露的 `bag_android_native_wrappers` object 产物，顶层 packaging target 不再直接持有 Android 私有 wrapper source list。
* 收紧 `tools/wavebits_tools/commands/retirement_policy.py` 的 post-legacy guard，`bag/legacy/**` 一旦以文件路径或 direct include token 形式回流就会被 `verify` 判定为失败。
* 重分组 `verify` 的 static policy，`bag/interface/common/*` 的存在性、allowed-owner、include-based 形态与 `bag.interface.*` module mirror 禁止规则改由 `compatibility` 组承接，`retirement` 组只保留 post-legacy / retired-surface / boundary-adjacent host wiring 语义。
* 重构 Android 与 host `--no-modules` baseline 的验证口径，固定 `python tools/run.py android native-debug` 与 `python tools/run.py android assemble-debug` 为 Android 独立 gate 的最小集合，同时把 `python tools/run.py verify --build-dir build/legacy-host --skip-android --no-modules` 收窄为纯 host retirement baseline 的待退出入口。
* 重构 host-only `--no-modules` baseline 的执行面，`verify` 在 `build/legacy-host` 下只保留 `unit_tests`、`api_tests`、`artifact_tests` 这 3 项最小 smoke，`cli_smoke_tests` 降级为 host 产品层 smoke，不再属于 baseline 最小集。
* 重构 `.github/workflows/verify-host-and-legacy.yml`、`docs/notes/legacy-release-gates.md`、`docs/testing.md`、`tools/README.md`、`docs/notes/build-commands.md` 与相关 architecture/temp 文档的正式口径，将 `verify --build-dir build/legacy-host --skip-android --no-modules` 从正式 CI / release baseline 降为本地 ad hoc 检查，并以显式 host C++17 replacement smoke（`configure --build-dir build/legacy-host --no-modules` + `build --build-dir build/legacy-host` + `test --build-dir build/legacy-host --tests-regex "^(unit_tests|api_tests|artifact_tests)$"`）完成替代定稿。
* 重构 `apps/audio_android/app/src/main/cpp/CMakeLists.txt` 的 JNI boundary 声明，保持 app CMake 继续只消费 `bag_api.h` 与 `native_package -> bag_android_native`，不回退到直接依赖 repo library targets 的旧拓扑。

### 修复 (Fixed)
* 修复总控文档把“总体目标”和“兼容偏差”混写的问题，当前总控主叙事只描述 module-first / host-side `import std;` 的终态，Android、`bag/internal/**`、`bag/legacy/**` 与 `--no-modules` 统一下沉到 `Current Deviations`。
* 修复 Android 与 root host `--no-modules` baseline 仍被当成同一职责的口径漂移，明确 Android 继续是 `C++17 + bag_api.h` 平台例外，而 `--no-modules` 只负责 host-side `bag/internal/**` non-modules declaration chain 的对照基线。
* 修复正式 workflow / docs 仍把 `verify --build-dir build/legacy-host --skip-android --no-modules` 当现役 baseline 的口径漂移，`docs/notes/legacy-retirement-preconditions.md` 中“`--no-modules` baseline 已正式下线或有明确替代方案”现已可标记为 `满足`。
* 修复 `Phase 1` 文档残留的 `bag/legacy/** declarations` 表述，统一改写为 `bag/internal/**` 支撑的 `C++17` internal declaration chain，避免将 dormant legacy carve-out surface 误写为现役实现依赖。
* 修复 `apps/audio_android/app/src/main/cpp/CMakeLists.txt` 缺失 `${WAVEBITS_ROOT}/libs/audio_api/include` / `bag_api` boundary token 导致 `verify` 静态检查失败的问题，并恢复 `build/dev verify`、`build/legacy-host verify`、`android native-debug` 与 `android assemble-debug` 的四条 gate 连通性。
* 修复 `bag_api -> transport.facade -> flash/pro/ultra` 仍被写成现役 fallback blocker 的口径漂移，明确 `libs/audio_api/src/bag_api.cpp` 与 `libs/audio_core/src/common/version.cpp`、`src/transport/transport.cpp`、`src/flash/*.cpp`、`src/pro/*.cpp`、`src/ultra/*.cpp` 已完成 modules-only cutover。
* 修复 host `WAVEBITS_HOST_MODULES=OFF` 与 `bag_api.cpp` fallback 责任仍被描述为待决策项的口径漂移，明确 root `CMakeLists.txt` 已直接退休 host `OFF`，Android focused gate 改为独立证明 `native_package -> bag_android_native` packaging lane。
* 修复 current-state docs / temp 控制文档仍把预留接口头记为主仓 `bag/internal/**` owner 的口径漂移，明确主仓 direct owner 已归零，预留接口头改由 `bag/interface/common/*` 保留独立声明层。
* 修复 current-state docs / release gates / tools 仍把 `bag/legacy/**` 写成 deletion-ready residual surface 的口径漂移，统一改为“已删除 + verify 阻止回流”的 post-legacy narrative。
* 修复 current-state docs / temp 控制文档仍把 `bag/interface/common/*` 写成待评估长期主题的口径漂移，统一改为“长期保留的 reserved-interface declaration boundary”，并明确这层不进入 host-only `import std;` 扩面。
