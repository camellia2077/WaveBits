## [v0.2.2] - 2026-03-12

### 新增功能 (Added)
* 新增 host-only `import std;` 试点开关与对应的 `WAVEBITS_CORE_IMPORT_STD` / `CXX_MODULE_STD` 接线。
* 新增标准库 `#include` inventory 文档，用于区分首批 pilot、扩面候选与暂缓区。
* 新增覆盖 `Test/modules/*`、`bag_core` 剩余实现 TU、核心导出 module interface、`bag.common.*`、直接消费者层与更外层 compatibility headers 的 import-std / standardization 静态门禁。

### 技术改进/重构 (Changed/Refactor)
* 重构 `Test/modules/*` 到真实 `import std;` 路径，并通过统一测试支持头收口 `u8` 字面量与 `char8_t` 兼容问题，目的是先在最可控的 host 路径上验证 `libs` 代码与标准库模块共存。
* 重构低风险 host 实现 TU 到 `import std;`，包括 `flash/pro/ultra codec.cpp`、`transport/compat/frame_codec.cpp`、`pipeline.cpp`、`transport.cpp`，目的是继续把 `libs/audio_core` 的实现层从传统标准库 `#include` 推向 module-first 形态。
* 重构 `flash/pro/ultra phy_clean.cpp`、`pro/ultra phy_compat.cpp`、`fsk/fsk_codec.cpp` 与对应 `.cppm`，扩展 host-only `import std;`，目的是把 `libs` modules 化从叶子实现推进到中层实现与对应 interface。
* 重构核心导出 module interface 到 `import std;`，覆盖 `flash/pro/ultra.codec`、`transport.compat.frame_codec`、`transport.facade`、`pipeline`，目的是让 `libs` 的核心 modules graph 在 host 路径下统一到 modules + `import std;` 口径。
* 重构 `bag.common.config` 与 `bag.common.types` 的 module interface / compatibility header spelling，目的是让 `libs` 基础层也能稳定服务 modules 路径，而不是继续停留在旧 typedef 口径。
* 重构 `bag.transport.decoder`、`bag.pipeline`、`bag.transport.facade`、`bag.transport.compat.frame_codec` 以及 `flash/pro/ultra` 更外层 bridge headers，收掉 module interface 与 header fallback 的 spelling 漂移，目的是继续缩小 `libs` 从 compatibility headers 消费能力的必要性。

### 修复 (Fixed)
* 修复 host 默认 modules 路径里 `char8_t` compatibility flags 与 `u8"..."` 测试语料的阻塞问题，使 `import std;` pilot 可在真实 host 构建中成立。
* 修复 `bag_core` 实现与 clang wrappers 在 `import std;` 路径下容易回退到旧 `<...>` include 形态的问题。
* 修复核心导出 interface、基础模块与外层 bridge headers 在 `std::uint8_t`、`std::size_t`、`std::int16_t` 等 spelling 上的长期漂移。
* 修复 `bag.common.* -> direct consumers -> outer headers` 这一整圈标准库 spelling 不一致的问题，为后续继续让 `libs` 代码靠近长期边界时仍保持 module-first 口径清路。
