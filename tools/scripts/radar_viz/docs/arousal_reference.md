# 维度参照标准：认知过载 (Arousal)

在 WaveBits 音频可视化体系中，"认知过载"不再单纯描述音量的刺耳程度，而是衡量**人类大脑在尝试解码/解析该信号时所承受的算力负担与精神压力**。

## 核心衡量指标：人类解析难度 (Human Parsing Difficulty)

该维度的数值由以下技术因素共同决定：

### 1. 传输速率 (Bit Duration)
*   **低过载 (1-3)**：信号极慢（如 `Litany` 的 6x 周期或 `Void` 的 2.5x 周期）。每一个 bit 的切换都清晰可辨，人类甚至可以用笔尖同步记录每一个 0 和 1。
*   **中过载 (4-6)**：标准速率（如 `Steady` 的 0.9375x）。节奏稳定，经过训练的解码员可以跟上其步进，但长时间监听会产生疲劳。
*   **高过载 (8-10)**：高速传输（如 `Hostile` 的 0.875x 或 `Ultra` 的 16-FSK）。信号密集程度超过了人类神经突触的反应极限。

### 2. 节奏稳定性 (Timing Variability)
*   **固定节奏**：即使速度稍快，只要节奏恒定，大脑就可以通过建立预期来降低解析难度。
*   **变速/变频 (`Zeal`)**：在传输途中突然改变 bit 窗口大小或载波频率。这种不确定性会强制大脑不断重置同步逻辑，导致"认知过载"迅速拉满。

### 3. 听觉压力 (Auditory Stress)
*   **纯净正弦波**：对神经系统友好。
*   **攻击性纹理 (`Hostile` / `Collapse`)**：引入硬剪切、点击音或震颤效果。这些声音在生物层面上会被识别为"威胁"或"错误"，从而在解码过程中引入额外的情绪负担。

---

## 典型参照案例：
*   **Litany (1)**：如同缓慢的诵经，解析难度极低。
*   **Ultra (4)**：虽然速度极快，但由于是纯净的 16-FSK 机器码且节奏极其规律，其"攻击性"反而不如一些狂乱的 Flash 模式。
*   **Zeal (10)**：狂热、变速、瞬态爆发，是人类脑算力的终极杀手。
